# System Design

설계 역량을 체계적으로 키우기 위한 학습 로그 저장소입니다.  
아키텍처 설계, 데이터 모델링, 인터페이스/API 설계, 운영·비기능 요구사항 등 **시스템 설계 전반에 대한 사고 과정과 기록**을 남기는 것을 목표로 합니다.

단순히 개념을 정리하는 것이 아니라, 실제 서비스를 설계한다는 가정하에  
“왜 이런 선택을 했는지, 어떤 대안을 버렸는지”까지 함께 기록합니다.

## Learning Scope

### 1. 아키텍처 설계 (Architecture Design)

- 『가상면접을 통해 배우는 대규모 시스템 설계 기초 1』을 중심으로 학습합니다.
- 요구사항 → 제약사항 → 아키텍처 의사결정의 흐름을 정리합니다.
- 주요 학습 주제
  - 모놀리식 vs 마이크로서비스, 계층형 아키텍처
  - DB / 캐시 / 메시지 큐 / 스토리지 등 컴포넌트 구성
  - 동기/비동기 통신, 확장성·가용성·일관성 트레이드오프
  - 장애/복구 전략, 옵저버빌리티(로그/메트릭/트레이싱) 기초

### 2. 데이터 모델링 (Data Modeling)

- 김영한의 『실전 데이터베이스 – 설계 1,2편, 현대적 데이터 모델링 완전 정복』을 중심으로 학습합니다.
- 논리/물리 모델을 구분하고, 도메인 관점에서 ERD를 설계합니다.
- 주요 학습 주제
  - 엔터티/속성/관계, 식별/비식별 관계
  - 정규화/비정규화, 이력·스냅샷·마스터 테이블 설계
  - FK·제약조건·인덱스 설계, 조회/통계용 모델링
  - 실제 도메인(예약, 배너, 정책, 사용자 등)을 재모델링하며 검증

## Logging Principles

- **사고 과정 중심**  
  - 최종 결과뿐 아니라, 고려했던 대안과 선택 근거를 함께 기록합니다.
- **일관된 형식 유지**  
  - 가능한 한 동일한 템플릿과 섹션을 사용해, 시간이 지나도 비교·회고하기 쉽게 만듭니다.
- **실제 서비스 관점**  
  - 추상적인 개념보다, “실제로 서비스에 도입한다면 어떻게 설계할지”를 우선합니다.
